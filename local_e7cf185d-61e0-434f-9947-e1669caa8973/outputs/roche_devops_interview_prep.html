🧠 Memory Aids & Mnemonics

📛 Mnemonic 1: GitOps with ArgoCD — "SDRD"

Remember the GitOps cycle: Store, Detect, Reconcile, Deploy
SStore — Everything lives in Git (manifests, configs, Helm charts)
DDetect — ArgoCD watches Git for changes
RReconcile — Argo compares desired (Git) vs actual (cluster)
DDeploy — Argo syncs the cluster to match Git
Say it: "ArgoCD does SDRD — it Stores in Git, Detects changes, Reconciles state, and Deploys."

📛 Mnemonic 2: Kubernetes Pod Troubleshooting — "DELD"

When a pod is broken, follow Describe, Events, Logs, Delete/redeploy
DDescribe — kubectl describe pod <name>
EEvents — check the Events section at the bottom of describe
LLogs — kubectl logs <pod> --previous if crashed
DDelete/fix — delete pod to let it restart, or fix the manifest

📛 Mnemonic 3: Terraform Workflow — "IPAPA"

Init, Plan, Apply, Push, Apply-again — the full IaC lifecycle
IInit — terraform init (downloads providers)
PPlan — terraform plan (shows what will change)
AApply — terraform apply (makes the change)
PPush — commit state to remote, PR to Git
AAutomate — wire it to CI/CD pipeline for next time

📛 Mnemonic 4: CI/CD Pipeline Stages — "BTTDD"

Build, Test, Tag, Deploy, Done
BBuild — compile code, build Docker image
TTest — run unit tests, linting, security scans
TTag — tag Docker image with commit SHA or version
DDeploy — push to registry, update k8s manifest, ArgoCD syncs
DDone — notify Slack, update status check on PR

"What's the biggest infrastructure challenge the team is facing right now that this role would help solve?" — Shows you want to understand real problems, not just pass the interview.

"How does the DevOps team collaborate with development squads — do you embed in teams, or work as a platform team?" — Shows interest in team dynamics and operating model.
